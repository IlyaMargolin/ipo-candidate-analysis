# Contributing

Thank you for your interest in improving this project.

## Contribution principles

Please keep contributions aligned with the repository's purpose:

- practical IPO candidate analysis;
- transparent methodology;
- reproducible workflows;
- readable documentation.

## Recommended contribution areas

- new scoring factors;
- sector-specific templates;
- better data validation;
- testing and error handling;
- improved documentation and examples.

## Contribution process

1. Fork the repository.
2. Create a feature branch.
3. Make focused, well-documented changes.
4. Submit a pull request with a clear explanation.

## Pull request checklist

Before opening a pull request, please confirm that:

- the notebook still runs end to end;
- new logic is documented;
- sample inputs and outputs remain understandable;
- comments and user-facing notes are in English.

## Style guidance

- Keep variable names descriptive.
- Add docstrings to reusable functions.
- Prefer transparent logic over unnecessary complexity.
- Document assumptions explicitly.
